# Togo – 移動予約確認アプリ
高速バス・飛行機などの移動予定を、視覚的でわかりやすく管理するシンプルな Web アプリです。

行き／帰りの都道府県・日付・予約状況をまとめて管理し、  
当日の予約確認 URL もすぐにアクセスできるように設計されています。

使ってみる: https://t0rixs.github.io/TransitMemory/

---

## ✨ 主な機能

### ■ 移動予定（Togo）の登録・編集
- 交通手段（飛行機 / 高速バス）  
- 行きの都道府県  
- 帰りの都道府県  
- 日付（カレンダー入力 + ダイヤル UI）  
- 予約ステータス（予約済み / 未予約）  
- 予約確認 URL の登録

👉 入力内容はすべてブラウザの LocalStorage に保存されるため、リロードしてもデータは保持されます。

---

### ■ 都道府県の「履歴学習」
過去に選択した都道府県を最大10件まで記録し、  
**セレクト欄の上位6件に優先表示**します。  
（--- 区切りの下には全47都道府県が表示されます）

これにより、よく使う区間を高速で選択できます。  
（例：徳島 ⇄ 東京を頻繁に使う人向け）

---

### ■ 今日の予約を自動でまとめて表示
今日の日付に該当する Togo がある場合、画面上部にカードとして強調表示されます。:contentReference[oaicite:0]{index=0}

- 予約済み → **確認 URL をワンクリックで開ける**
- 未予約 → **赤色の警告 UI で注意喚起**

旅行当日の「どこで予約確認するんだっけ？」をゼロにします。

---

### ■ 時系列での表示 + 接続線 + 滞在日数
Togo は自動で日付順に並び替えられ、  
連続する区間の場合は **「接続線」と「滞在日数」** が表示されます。:contentReference[oaicite:1]{index=1}

例：
徳島 → 東京（1/3）
　↓ 東京で2日滞在
東京 → 札幌（1/5）


旅程のつながりが一目で把握できます。

---

### ■ 予約ステータス管理（ライト UI）
各 Togo には「予約済み / 未予約」を示すライトを搭載。  
クリックで状態を切り替え、未予約なら URL 入力モーダルを開きます。:contentReference[oaicite:2]{index=2}

---

### ■ 期限切れデータの自動削除
今日より過去の Togo は自動的に削除されます。  
古い旅程が残り続けないため、いつも見やすい状態が保たれます。

---

### ■ UI・デザイン
- カード型 UI  
- カラフルな予約ステータス  
- スマホ対応レスポンシブデザイン  
- ダイヤル UI 付き日付選択  
- モーダル操作、クリック外閉じ、ESC 閉じにも対応

CSS はすべてカスタムデザインです。:contentReference[oaicite:3]{index=3}

## 🧭 想定ユースケース

### ● 学生の帰省・旅行の管理
- 徳島 → 東京  
- 東京 → 北海道  
など、複数区間の旅程を視覚的に整理できます。

### ● バス・飛行機をよく使う人の予約忘れ防止
- 今日の予約をトップに表示  
- 1週間以内未予約なら警告表示  

「予約確認 URL がメールのどこかに埋もれる問題」を解消。

### ● 地方と都市を往復する生活者
毎週・毎月決まった区間を使う人は  
履歴優先表示で爆速入力できます。

### ● 出張の管理
複数区間を含む出張のスケジュール可視化に最適。

## 🛠 使用技術
- HTML / CSS / JavaScript  
- LocalStorage を用いたデータ永続化  
- モーダル UI  
- カスタム日付ダイヤル UI

## 🚀 セットアップ方法

ローカルで動かす場合：

git clone <repository-url>
cd togo-app
open index.html # あるいはブラウザで開く


サーバー不要・完全にフロントだけで動きます。
